# 📈 StockManager — 미국 주식 장기/스윙 투자 관리 워크벤치

엑셀 스코어카드를 그대로 디지털화하고, 월스트리트식 밸류에이션(정/역 DCF, 멀티플)과
실시간 시세 연동, 촉매(호재/악재) 반영 추적, 매매기록·손익까지 한 화면에서 관리하는
개인용 투자 분석 프로그램.

## 실행 방법

```
# 1) 최초 1회 패키지 설치
pip install -r requirements.txt

# 2) 실행 (둘 중 하나)
run.bat                      # 더블클릭 (Windows)
streamlit run app.py         # 터미널
```

실행하면 브라우저에서 `http://localhost:8501` 이 자동으로 열립니다.

> ⚠️ **업데이트 후 이상한 오류(예: `AttributeError`)가 나면 브라우저 새로고침이 아니라
> 터미널에서 `Ctrl+C`로 완전히 끄고 다시 실행**하세요. Streamlit은 `app.py` 변경은 자동
> 반영하지만, `storage.py` 등 다른 파일이 바뀐 건 프로세스를 껐다 켜야 반영됩니다.

## 주요 기능

### 📊 대시보드
- 등록 종목 전체의 현재가·등락률·**30일 추이(스파크라인)**·스코어·매수판정·보유손익을 한 표에서.
- 점수 ≥70 매수 / 60~70 소액 / <60 보류 (사용자 규칙 자동 판정).
- **표에서 종목 행을 클릭하면 해당 종목 상세로 바로 이동.**

### 🔎 스캐너
- 워치리스트 전체를 한 번에 스캔: 점수·판정·Forward P/E·현재 추세(일봉 SMA).
- **추세 전환 알림**: 최근 5봉 이내 골든/데드크로스 발생 종목을 상단 배너로 경고.

### 🔍 종목 상세 (7개 탭)
1. **개요/Live** — Robinhood 스타일 차트가 상단에 크게:
   - **✨ 다이나믹 차트(TradingView)**: 기본이 증권앱 느낌의 부드러운 실시간 렌더링
     (streamlit-lightweight-charts-v5). 끄면 기존 분석용 plotly 차트로 전환(추세/촉매 로직은
     두 렌더러가 동일하게 공유).
   - **차트 종류**: 라인(영역) · **일봉 · 주봉 · 월봉**(캔들+SMA) 전환.
   - **기간 pill**: 1D/1W/1M/3M/1Y/5Y … **10Y·MAX**(20년+ 전체 히스토리)까지 종류별 적정 범위.
   - **📊 차트 분석 버튼**: 5·20·60·120일선 기준 현재 추세 진단(정배열/역배열/혼조),
     이격도·최근 골든/데드크로스·쉬운 요약 + "이동평균선 읽는 법" 가이드 팝업.
   - **🔴 실시간 토글**: 30초마다 차트만 자동 갱신(전체 새로고침 없이, `st.fragment`).
   - **추세 전환 자동 탐지**: 단/장기 SMA 크로스오버로 🟩상승·🟥하락 추세 구간을 음영
     표시하고 ▲/▼ 전환점 마커 표시.
   - **내러티브 오버레이**: 내가 '촉매 반영' 탭에 기록한 호재/악재(★)를 해당 날짜의
     가격 위에 표시(호버로 헤드라인·메모·반영도 확인). 차트 아래 **"추세 전환 구간 &
     내러티브 펼쳐보기"** 에서 각 전환점과 인접한 뉴스/내러티브를 묶어서 확인.
   - 그 아래 **Fundamental** 섹션(전 지표 영문): Trailing/Forward P/E, P/B, P/S, EV/EBITDA,
     PEG, Short Interest(%Float), Beta, Revenue/Earnings Growth, Profit Margin, Net Debt,
     FCF(TTM), EPS(TTM/Fwd), Debt/Equity, 52-Week Range. 옆에 **📖 Guidebook**(주린이용
     지표 해설) 버튼. 각 섹션마다 **❔ Guide** 팝업으로 읽는 법 설명.
   - **📊 거래량 패널 토글**: 차트 하단에 상승/하락 색상 거래량 막대.
2. **스코어카드** — 첨부 엑셀을 그대로 재현. Valuation(33%)·Risk(20%)·Leader(12%)·
   Technology(35%) 카테고리. 가중치·점수·코멘트 직접 수정, 가중 총점·레이더차트·판정.
   **Live 실측 자동 채움** — PER·PSR·PBR·CAGR(매출/EPS)·FCF를 Yahoo에서 읽어 각 항목 옆에
   자동 표시(읽기전용). 점수(0~100)는 이 값을 참고해 본인이 매깁니다.
3. **밸류에이션·DCF** — 멀티플 스냅샷, Forward P/E 향후 N년 투영, **정 DCF 내재가치 &
   안전마진**, **역 DCF(Reverse DCF)** — 현재 시총이 가정하는 시장 내재 성장률을 역산해
   "현재가가 비싼지/싼지"를 헤지펀드식으로 판정. **동종업계 멀티플 비교** — 경쟁사 티커를
   입력하면 Fwd P/E·PBR·PSR·EV/EBITDA를 나란히 비교하고 피어 중앙값 대비 저/고평가 판정.
4. **🎯 시나리오** — Bear·Base·Bull·Super Bull(+케이스 추가) 케이스별로 EPS·매출·**사용자지정
   지표**(회원수·계약건수·판매량·결제금액 등, ➕지표추가)의 **연평균성장률(CAGR)을 슬라이더
   (−100~+100%)** 로 3년/5년 가정. EPS×exit P/E, 매출×P/S로 **목표주가·업사이드**, 케이스별
   **코멘트**, 확률가중 **기대 목표주가(blended)** 와 케이스 비교 막대차트.
5. **📰 뉴스·Narrative** — "시장이 이 종목을 어떤 이야기로 보는가"를 무료 데이터로 근사.
   **뉴스 버즈**(0~100, 노출량) · **헤드라인 톤**(-100~+100, 감성분석+금융어휘 보강) ·
   **애널리스트 모멘텀**(최근 수개월 컨센서스가 강세/약세로 이동 중인지) · **Google 검색
   관심도**(12개월, 실패 시 조용히 숨김). 뉴스 카드에서 **⭐ 촉매로 기록** 버튼으로 원클릭
   저장. (무료 API 한계상 "과거 대비 z-score"는 불가 — 현재 시점 스냅샷이라는 점을 가이드에 명시)
6. **촉매 반영** — 호재/악재 이슈를 날짜·기대영향·**현재가 반영도(%)**로 기록·추적.
   반영도 50% 미만 = 아직 덜 반영된 촉매 알림.
7. **매매기록** — BUY/SELL 기록, 평균단가·평가액·실시간 평가손익 자동 계산.

## 실시간 데이터 (가능 여부 확인 완료 ✅)

- **데이터 소스**: Yahoo Finance (yfinance, 무료, API 키 불필요).
- **시세**: 일반장 기준 **실시간 ~ 약 15분 지연**(거래소별 상이). 새로고침 시 갱신.
- 시세 캐시: 현재가 60초 / 펀더멘털 5분 / 차트 10분 / 뉴스 15분 (자동). 사이드바
  **🔄 시세 새로고침** 버튼으로 즉시 갱신.
- **진짜 틱 단위 실시간**(스트리밍)이 필요하면 유료 API(Polygon, Alpaca, IEX Cloud)가
  필요합니다. 개인 분석·기록 용도에는 현재 무료 소스로 충분합니다.

## 데이터 저장 & 계정
- **회원가입/로그인**: 이름+비밀번호로 계정을 만듭니다. 비밀번호는 **scrypt로 해시**해
  저장되고(`users` 테이블), 로그인 시 실제로 검증됩니다 — 오타로 새 빈 계정이 생기지 않습니다.
  (※ 이메일 찾기 등 완전한 보안 인증은 아닙니다 — 민감정보는 입력하지 마세요.)
- **저장 백엔드는 두 가지, 자동 전환**:
  - 기본(설정 없음): **SQLite** — 계정별 파일(`data/portfolio_<해시>.db`)로 격리. 로컬 실행엔
    충분하지만, 무료 클라우드 재배포 시 **휘발**될 수 있습니다.
  - `DATABASE_URL`(무료 Neon Postgres 등)을 넣으면 **자동으로 Postgres 백엔드**로 전환 —
    계정별 **스키마**로 격리되고, 재배포/재시작에도 데이터가 **영구 보존**됩니다. (아래 배포
    가이드의 3번 참고)
- SQLite 모드에선 사이드바 **"👤 계정 / 데이터 백업"** 에서 `.db` 백업(다운로드)/복원(업로드)
  가능(Postgres 모드는 이미 영구 저장이라 이 버튼은 비활성).

## ☁️ 친구와 함께 쓰기 — 웹 배포 (Streamlit Community Cloud, 무료)

설치 없이 **링크 하나로** 친구도 쓰게 하는 방법. (약 15분, 신용카드 불필요)

1. **GitHub 준비**: [github.com](https://github.com) 가입 → 새 저장소(repository) 생성 →
   이 폴더를 올립니다(로컬에 이미 `git init` + 첫 커밋이 돼 있다면 3~4번만 실행).
   ```
   cd StockManager
   git branch -M main
   git remote add origin https://github.com/<내계정>/<저장소>.git
   git push -u origin main
   ```
   (`.gitignore`가 `*.db`·`data/`·`.streamlit/secrets.toml`을 자동 제외하므로 개인 기록·
   비밀값은 올라가지 않습니다.)
2. **(선택, 강력 추천) Neon 무료 Postgres로 영구 저장 설정**:
   - [neon.tech](https://neon.tech) 접속 → GitHub/Google로 가입(무료, 카드 불필요) →
     **New Project** 생성.
   - 프로젝트 생성 후 뜨는 **Connection string**(`postgresql://...`)을 복사해둡니다.
   - 이 값은 3번 Streamlit Cloud의 **secrets**에 붙여넣을 것이므로 지금은 복사만 해두세요.
3. **배포**: [share.streamlit.io](https://share.streamlit.io) 접속 → GitHub로 로그인 →
   **New app** → 저장소/브랜치(main)/메인 파일 `app.py` 선택.
   - **Advanced settings → Secrets** 에 아래처럼 입력(2번에서 복사한 값으로 교체):
     ```toml
     DATABASE_URL = "postgresql://사용자:비번@호스트/디비이름?sslmode=require"
     ```
   - **Deploy** 클릭. (`requirements.txt`로 패키지 자동 설치, 1~3분 소요.)
4. **공유**: 생성된 `https://<앱이름>.streamlit.app` 링크를 친구에게 전달.
   각자 회원가입 탭에서 자기 이름+비밀번호로 계정을 만들면 데이터가 분리·영구 저장됩니다.

### ⚠️ 배포 시 꼭 알아둘 점
- **Neon(2번)을 건너뛰면** SQLite 모드로 배포되며, Streamlit 무료 클라우드 특성상 재시작 시
  기록이 초기화될 수 있습니다 — 이 경우 사이드바 백업 버튼을 주기적으로 쓰세요.
- **Neon을 설정하면** 이 문제가 사라집니다(무료 티어 0.5GB로 개인/친구용엔 충분).
- 기존 사용자(이 프로젝트를 로컬에서 먼저 써봤다면)는 **예전과 동일한 이름·비밀번호로
  회원가입 탭에서 새로 계정을 만들면** 기존 데이터가 자동으로 이어집니다(별도 이전 작업 불필요).
- Python 3.10 이상 권장(클라우드 기본값으로 동작).

> **추세 전환 + 내러티브 작동 원리**: 과거 임의 시점의 뉴스는 무료 API로 받기 어렵기
> 때문에, 추세 전환점(자동 탐지)에 **내가 기록한 촉매**를 연결하는 방식입니다. "📰 뉴스·
> Narrative" 탭에서 마음에 드는 뉴스를 ⭐ 원클릭으로 촉매에 담으면, 차트의 ★ 마커로 나타나
> "이 하락 구간에 무슨 일이 있었나"를 바로 펼쳐볼 수 있습니다.
