## 1. 이 프로젝트가 뭔가
미국 주식 **장기/스윙 투자 관리 & 밸류에이션 워크벤치**.
사용자의 엑셀 스코어카드(`주식 엑셀.png`)를 디지털화 + 월스트리트식 밸류에이션 + 실시간
시세 + Robinhood UI + 추세/내러티브 분석 + 친구와 웹 공유까지 구현한 Streamlit 웹앱.

- **위치**: `C:\Users\USER\Desktop\Claude\StockManager`
- **실행**: `run.bat` 더블클릭  또는  `streamlit run app.py` → 브라우저 `localhost:8501`
- **환경**: Windows 11, Python 3.12 (node 없음), 콘솔 출력은 UTF-8 필요(`PYTHONIOENCODING=utf-8`)

## 2. 기술 스택 / 데이터
- Python + **Streamlit 1.58** UI, **plotly** 차트, **pandas**.
- 시세: **yfinance (Yahoo Finance, 무료·API키 불필요)**. 실시간~15분 지연. API 키 없음.
- 저장: **SQLite(기본) / Postgres(Neon, `DATABASE_URL` 설정 시 자동전환)** 이중 백엔드.
  SQLite는 사용자별 파일(`data/portfolio_<해시>.db`), Postgres는 사용자별 **스키마**로 격리.
- 인증: 이름+비밀번호 **scrypt 해시**(`users` 테이블, stdlib `hashlib.scrypt`) — 진짜 검증됨.

### 4-3. 2026-07-02 Phase 1 — 영구 저장 + 진짜 로그인
- ✅ **Postgres(Neon) 이중 백엔드**(`storage.py`): `DATABASE_URL` 미설정 시 기존 SQLite 그대로
  (로컬 개발·테스트 무변경). 설정되면 자동으로 Postgres 사용 — **사용자별 Postgres 스키마**로
  격리(`CREATE SCHEMA "u_<해시>"` + `SET search_path`)해서 모든 기존 SQL(`?` 플레이스홀더,
  테이블명 등)이 거의 그대로 재사용됨(`_PGCursorWrapper`가 `?`→`%s` 치환). `AUTOINCREMENT`만
  Postgres용 `SERIAL`로 별도 스키마 문자열 분리(`_SCHEMA_SQLITE` / `_SCHEMA_PG`).
- ✅ **scrypt 해시 로그인**: `users` 테이블(고정 위치 — SQLite는 `data/_auth.db`, Postgres는
  `public` 스키마) + `create_user`/`verify_password`/`user_exists`. 앱 게이트가 로그인/회원가입
  탭으로 분리, 오답 비밀번호는 명확히 거부(예전엔 오타로도 새 빈 계정이 조용히 생성됐음).
- ✅ **기존 사용자 데이터 100% 보존**: 계정별 데이터 격리 키(`user_db_path`) 공식을 **그대로
  유지**(`hash(이름.lower()+"::"+비밀번호)`) — 새 `users` 인증 테이블은 이 키와 별개로 추가된
  것뿐이라, 기존 사용자가 예전과 같은 이름+비번으로 "회원가입"만 다시 하면 기존 파일에 자동
  재연결됨(별도 마이그레이션 스크립트 불필요, 자연스러운 흐름으로 해결).
- ✅ **git 저장소 초기화 + 첫 커밋** 완료(`ad74df0`). `.gitignore`가 `*.db`/`data/`/
  `.streamlit/secrets.toml` 제외 확인함(민감정보 미포함). git identity는 사용자가 직접
  `git config --global`로 설정(Gwangsu / an78964@naver.com) — 이 저장소 안내에 따라 진행.
- ⚠️ **알려진 한계**: `export_db_bytes`/`import_db_bytes`(사이드바 백업 버튼)는 SQLite 파일
  직접 백업 방식이라 Postgres 모드에선 비활성(둘 다 조용히 no-op) — Postgres는 이미 영구
  저장이라 필요성 낮음. 정말 필요해지면 pg_dump 또는 JSON 전체 내보내기로 별도 구현 필요.
- ✅ **[2026-07-02 실배포 후 수정] Postgres SyntaxError 실전 버그**: 실제 Streamlit Cloud +
  Neon 배포 후 종목 추가 시 `psycopg2.errors.SyntaxError` 발생 — 원인은 `add_ticker()`의
  `INSERT OR IGNORE INTO ...`가 **SQLite 전용 문법**이라 Postgres에서 그대로 깨짐(로컬은
  SQLite로만 테스트해서 못 잡았던 케이스). `INSERT INTO ... ON CONFLICT(symbol) DO NOTHING`
  (SQLite 3.24+/Postgres 둘 다 지원, 파일 내 다른 UPSERT들과 동일 패턴)으로 수정.
  재발 방지로 `test_storage_sql_no_sqlite_only_syntax`(SQL 문자열에 `INSERT OR`/`REPLACE INTO`
  같은 SQLite 전용 구문이 없는지 정적 검사) + `test_add_ticker_idempotent` 추가.
  **교훈**: mock 기반 PG 어댑터 테스트로는 실제 SQL 문법 오류를 못 잡는다 — 가능하면 실
  Neon(무료)에 한 번씩 직접 붙여보는 게 확실함.

  `session_state["_just_logged_in"]` 플래그를 세워, 로그인 게이트 통과 직후 렌더링되는
  메인 페이지(`div[data-testid="stAppViewContainer"]`)에 0.6초 페이드인 애니메이션을 1회만
  적용(플래그를 즉시 pop하므로 이후 평범한 rerun에는 재생 안 됨) — 오버레이가 사라진 자리에
  메인 화면이 살짝 떠오르듯 나타나는 전환감을 만듦.
  - 가정 기록: f-string 템플릿에 `.format(cls=...)`을 쓰면 이미 평가되어 단일 중괄호가 된
    CSS 블록과 충돌해 크래시하므로, `%%CLS%%` 플레인 텍스트 placeholder + `str.replace()`로
    치환(포맷 충돌 회피). Streamlit은 SPA가 아니라 rerun마다 전체 재실행이라 완벽한 크로스
    페이드는 불가능 — "오버레이 페이드아웃 → 메인 페이드인" 2단계로 근사함.
- 검증: Playwright로 회원가입 직후 여러 시점(초반/중반/페이드아웃 직전/최종 대시보드)
  스크린샷 4장 촬영 — 로켓이 path를 따라 이동하며 기울어짐, 차트가 화면 상단까지 뚫고
  나가는 급등 형태, 초록 화살촉, "LET'S GET RICH 🚀" 펄스, 대시보드 최종 렌더까지 확인.
  전체 테스트(20/8/11/5+smoke) GREEN 유지. 테스트 계정 정리 완료.

### 4-10. 2026-07-04 UI/UX 리디자인 5차 — 모바일 대응 + 로딩 화면 전체화면 확대 + 3초 고정
사용자 요청: 친구들이 폰으로도 접속 가능하게, 로딩 애니메이션의 차트가 화면 왼쪽아래~
오른쪽위 80%를 채우도록 키우고, 로켓이 차트를 따라가며 화성에 도달하게, 로딩을 총 3초로.
- ✅ **모바일 접속**: 이미 Streamlit Cloud 배포 URL 자체가 표준 반응형 웹앱이라 폰 브라우저
  에서 별도 설치 없이 그대로 접속 가능(Streamlit 기본 HTML에 viewport meta 태그 포함,
  좁은 화면에서 사이드바가 자동으로 접히는 서랍(햄버거) 형태로 전환됨 — 프레임워크가 이미
  제공하는 반응형 동작). 우리가 추가한 커스텀 CSS가 이를 깨지 않는지 Playwright의 iPhone 13
  디바이스 에뮬레이션으로 검증(로그인 게이트/로딩 화면/대시보드 3곳 모두 정상 렌더 확인).
  가독성 다듬기용 `@media (max-width: 640px)` 규칙 추가(사이드바 지수 위젯 폰트/여백 축소,
  h1 크기 축소, 로딩화면 타이틀·stage 크기 축소).
- ✅ **로딩 차트 전체화면 확대**: 기존엔 차트가 오버레이 중앙의 작은 고정 박스(760×320px)에
  갇혀 화면 왼쪽아래~오른쪽위 대부분이 비어 보였음. `<div class="sm-loading-stage">`(88vw×
  64vh, 화면의 대부분을 차지)로 감싸고 그 안에서 SVG가 `width:100%; height:100%;
  preserveAspectRatio="none"`로 stage를 꽉 채우도록 재구성. viewBox를 1200×800(percentage
  형태) 좌표계로 바꿔 path가 stage 전체 대각선(왼쪽 아래 x=40,y=760 → 오른쪽 위 x=1080,y=60,
  즉 스테이지 폭·높이의 약 87%씩 사용)을 가로지르게 재설계.
- ✅ **로켓이 화성에 도착**: 기존엔 로켓/추세선 애니메이션이 `repeatCount="indefinite"`로
  무한 루프했는데, 로딩이 3초짜리 단발성 연출로 바뀌면서 "도착"의 의미가 없어짐 →
  `repeatCount` 제거(SMIL 기본값 1회) + `fill="freeze"`로 바꿔 로켓이 path 끝(=화성 표면
  근처, `sm-loading-planet`을 path 종점과 같은 좌표계 비율로 배치)에 도달한 채 정지하도록
  변경. 화살촉(`marker-end`)이 애니메이션 초반부터 미리 보여 결말을 스포하던 문제도
  발견해 `opacity:0`→`animation-delay`로 로켓 도착 직전(2.05s)에만 나타나도록 수정.
- ✅ **총 3.0초 고정**: `_play_login_loading()`의 두 `time.sleep()` 합을 2.6+0.4=3.0초로
  정확히 맞추고, CSS/SMIL 애니메이션 duration도 2.3초(비행)+정지+0.4초(페이드아웃)로 그
  안에 맞아떨어지게 재조정(이전엔 무한루프 + 2.1초 sleep이라 실제 체감 길이가 불명확했음).
  - 가정 기록: 첫 검증 라운드에서 오버레이가 ~4.9초까지 떠 있는 것처럼 보여 조사했으나,
    server-side timing 로그(버퍼링 이슈로 로그 자체는 못 얻었지만) 및 재측정 결과 재현되지
    않아 콜드스타트(서버 막 재시작 직후 첫 요청) 변동으로 판단 — 이후 측정에서는 코드대로
    ~3.0초(2.85s에 오버레이 있음 → 3.81s에 사라짐)로 확인됨. 재발 시 서버 프로세스
    warm-up 여부를 먼저 의심할 것.
- 검증: Playwright로 (1) 데스크톱 1600×950에서 로딩 3프레임(초반/중반/도착) — 차트가
  화면 왼쪽아래부터 오른쪽 위 화성까지 거의 꽉 채우며 로켓이 화살표와 함께 화성에 도달하는
  것 확인 (2) iPhone 13 에뮬레이션으로 게이트/로딩/대시보드 3장 — 반응형 레이아웃과 로딩
  화면 모두 폰 화면에 맞게 정상 렌더 확인. 전체 테스트(20/8/11/5+smoke) GREEN 유지.

## 6. 알려진 한계 / 기술부채
- `use_container_width` deprecation 경고(콘솔만, 동작 정상). streamlit 1.58에선 작동.
  추후 `width='stretch'`로 교체 가능.
- **SQLite 모드**(Neon 미설정 시)는 여전히 Streamlit Cloud 재시작 시 휘발 가능 →
  Neon 설정을 강력 권장(README 배포 가이드 2번). 설정하면 이 문제 해결됨.
- **Narrative 버즈/톤은 현재 시점 스냅샷**(위 4-4 참조) — 무료 API 한계로 과거 시계열 대비
  z-score는 계산 불가. 방향성 참고용으로만 사용.
- 과거 임의시점 뉴스는 무료 API로 못 받음 → "추세전환+내러티브"는 **사용자가 기록한 촉매**를
  전환점에 연결하는 방식(의도된 설계). 이제 뉴스 카드에서 원클릭으로 촉매 기록 가능(4-4).
- 사용자 게이트는 **scrypt 해시로 실제 검증**하지만 이메일 찾기·2FA 등 완전한 보안 인증은
  아님. 민감정보 입력 금지 원칙은 유지.
- 루트의 `portfolio.db`는 초기 테스트 잔재(gitignore됨). 신규 구조는 `data/` 사용.
- Postgres 백업/복원 미지원(위 4-3 참조).
- ⚠️ **`storage.py`처럼 import된 모듈을 수정한 뒤엔 브라우저 새로고침이 아니라 streamlit
  프로세스 자체를 재시작해야 반영됨.** Streamlit 파일감시는 `app.py`(메인) 재실행은
  자동으로 해주지만, `import storage as ST` 같은 서브모듈은 `sys.modules`에 캐시돼 있어서
  코드만 바뀌어선 재로드되지 않음(`AttributeError: module 'storage' has no attribute
  'DATABASE_URL'` 같은 오류로 나타남 — 2026-07-02 실제 발생·확인). 터미널에서 `Ctrl+C` →
  `run.bat` 재실행하면 해결.

## 7-1. 데이터 모델 메모(신규)
- `scenarios` 테이블: symbol PK, JSON `{horizon_years, cases:[{name,color,prob,comment,
  drivers:[{key,kind:eps|revenue|custom,base,cagr,unit,exit_multiple,_id}], _id}]}`.

- `users` 테이블(고정 위치 — 사용자 스키마/파일과 별개): name PK, pw_hash, pw_salt, created_at.
