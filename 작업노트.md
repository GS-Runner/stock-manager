## 1. 이 프로젝트가 뭔가
미국 주식 **장기/스윙 투자 관리 & 밸류에이션 워크벤치**.
사용자의 엑셀 스코어카드(`주식 엑셀.png`)를 디지털화 + 월스트리트식 밸류에이션 + 실시간
시세 + Robinhood UI + 추세/내러티브 분석 + 친구와 웹 공유까지 구현한 Streamlit 웹앱.

- **위치**: `C:\Users\USER\Desktop\Claude\StockManager`
- **실행**: `run.bat` 더블클릭  또는  `streamlit run app.py` → 브라우저 `localhost:8501`
- **환경**: Windows 11, Python 3.12 (node 없음), 콘솔 출력은 UTF-8 필요(`PYTHONIOENCODING=utf-8`)

## 2. 기술 스택 / 데이터
- Python + **Streamlit 1.58** UI, **plotly** 차트, **pandas**.
- 시세: **yfinance (Yahoo Finance, 무료·API키 불필요)**. 실시간~15분 지연. API 키 없음.
- 저장: **SQLite**. 사용자별 격리 DB = `data/portfolio_<해시>.db`.

## 4. 구현 완료 기능 (요청 → 구현)
- ✅ **엑셀 스코어카드** 재현: 카테고리/가중치/내점수/코멘트 편집, 가중총점, 레이더차트,
  매수판정(70↑매수/60~70소액/60↓보류), Live지표 기반 자동채점 *제안*.
- ✅ **역DCF(Reverse DCF)**: 현재 시총을 정당화하는 시장 내재 성장률 역산 → 비싼지/싼지 판정.
- ✅ **밸류에이션**: PER, 향후 N년 Forward P/E 투영, PBR, PSR, EV/EBITDA, PEG, 순부채.
- ✅ **촉매(호재/악재) 반영 추적**: 날짜·기대영향·**현재가 반영도(%)** 기록, Yahoo 뉴스 표시.
- ✅ **매매기록/손익**: BUY/SELL, 평단·실시간 평가손익.
- ✅ **실시간 라이브 차트**: `st.fragment(run_every="30s")` 차트만 자동갱신.
- ✅ **Robinhood UI**: 다크테마 + 그린/레드 영역라인 + 세그먼트 pill 컨트롤.
- ✅ **일/주/월봉 + 추세전환 구간 + 내러티브**: SMA 크로스오버로 상승/하락 구간 음영,
  ▲/▼ 전환점 마커, 내가 기록한 촉매(★)를 해당 날짜에 오버레이 + "펼쳐보기"로 묶어보기.
- ✅ **웹 배포(멀티유저)**: 이름+비번 사용자 격리 + 데이터 백업/복원 + Streamlit Cloud 가이드.

### 4-1. 2026-07-01 추가 기능
- ✅ **시나리오/케이스 빌더**(종목상세 🎯 탭): Bear/Base/Bull/Super Bull(+케이스 추가/삭제).
  케이스별 EPS·Revenue·**사용자지정 지표**(회원수/계약건수/판매량/결제금액 등, ➕지표추가) 를
  **CAGR 슬라이더(−100~+100%)** 로 3년/5년 투영. EPS×exit P/E, Revenue×P/S로 **목표주가·업사이드**,
  케이스별 **코멘트**, 확률가중 **blended 기대 목표주가** + 케이스 비교 막대차트. Live값으로 기본 4케이스 자동생성.
- ✅ **스코어카드 Live 자동 채움**: PER/PSR/PBR/CAGR(매출·EPS)/FCF 를 Yahoo에서 읽어 'Live 실측' 열
  자동 표시(읽기전용, 저장 제외·매 로드 재계산). 점수(0~100)는 여전히 본인 주관.
- ✅ **거래량 패널**: 차트 하단 별도 y축 거래량 막대(토글). 상승/하락 봉 색 구분.
- ✅ **동종업계 PER 비교**(밸류에이션 탭): 경쟁사 티커 입력→ Fwd P/E·PBR·PSR·EV/EBITDA 표 +
  피어 중앙값 대비 저/고평가 판정 + 막대차트. 피어 목록 저장(peers 테이블).
- ✅ **엑셀 내보내기**(사이드바): 스코어카드·촉매·매매·시나리오·워치리스트를 .xlsx 다중시트로.
- ✅ **워치리스트 스캐너**(🔎 페이지) + **추세전환 알림**: 전체 종목 점수·판정·Fwd P/E·현재추세,
  최근 5봉 이내 골든/데드크로스 발생 시 상단 배너 알림.

## 6. 알려진 한계 / 기술부채
- `use_container_width` deprecation 경고(콘솔만, 동작 정상). streamlit 1.58에선 작동.
  추후 `width='stretch'`로 교체 가능.
- **Streamlit Cloud 무료 = 저장소 휘발성.** 재시작 시 데이터 초기화 가능 → 백업버튼으로 대비.
  자동·영구저장 원하면 무료 Postgres(Neon) 연동 필요(미구현).
- 과거 임의시점 뉴스는 무료 API로 못 받음 → "추세전환+내러티브"는 **사용자가 기록한 촉매**를
  전환점에 연결하는 방식(의도된 설계).
- 사용자 게이트는 **진짜 인증 아님**(단순 격리). 민감정보 입력 금지.
- 루트의 `portfolio.db`는 초기 테스트 잔재(gitignore됨). 신규 구조는 `data/` 사용.

## 7-1. 데이터 모델 메모(신규)
- `scenarios` 테이블: symbol PK, JSON `{horizon_years, cases:[{name,color,prob,comment,
  drivers:[{key,kind:eps|revenue|custom,base,cagr,unit,exit_multiple,_id}], _id}]}`.
  `_id`(uuid)는 Streamlit 위젯 키 안정화용 — 저장돼도 무해, 재계산 파생.
- `peers` 테이블: symbol PK, JSON 티커 배열.
- 스코어카드의 `metric`(Live 실측)은 **저장 안 함** — autofill_metrics로 매 로드 재계산.
