## 1. 이 프로젝트가 뭔가
미국 주식 **장기/스윙 투자 관리 & 밸류에이션 워크벤치**.
사용자의 엑셀 스코어카드(`주식 엑셀.png`)를 디지털화 + 월스트리트식 밸류에이션 + 실시간
시세 + Robinhood UI + 추세/내러티브 분석 + 친구와 웹 공유까지 구현한 Streamlit 웹앱.

- **위치**: `C:\Users\USER\Desktop\Claude\StockManager`
- **실행**: `run.bat` 더블클릭  또는  `streamlit run app.py` → 브라우저 `localhost:8501`
- **환경**: Windows 11, Python 3.12 (node 없음), 콘솔 출력은 UTF-8 필요(`PYTHONIOENCODING=utf-8`)

## 2. 기술 스택 / 데이터
- Python + **Streamlit 1.58** UI, **plotly** 차트, **pandas**.
- 시세: **yfinance (Yahoo Finance, 무료·API키 불필요)**. 실시간~15분 지연. API 키 없음.
- 저장: **SQLite(기본) / Postgres(Neon, `DATABASE_URL` 설정 시 자동전환)** 이중 백엔드.
  SQLite는 사용자별 파일(`data/portfolio_<해시>.db`), Postgres는 사용자별 **스키마**로 격리.
- 인증: 이름+비밀번호 **scrypt 해시**(`users` 테이블, stdlib `hashlib.scrypt`) — 진짜 검증됨.

### 4-3. 2026-07-02 Phase 1 — 영구 저장 + 진짜 로그인
- ✅ **Postgres(Neon) 이중 백엔드**(`storage.py`): `DATABASE_URL` 미설정 시 기존 SQLite 그대로
  (로컬 개발·테스트 무변경). 설정되면 자동으로 Postgres 사용 — **사용자별 Postgres 스키마**로
  격리(`CREATE SCHEMA "u_<해시>"` + `SET search_path`)해서 모든 기존 SQL(`?` 플레이스홀더,
  테이블명 등)이 거의 그대로 재사용됨(`_PGCursorWrapper`가 `?`→`%s` 치환). `AUTOINCREMENT`만
  Postgres용 `SERIAL`로 별도 스키마 문자열 분리(`_SCHEMA_SQLITE` / `_SCHEMA_PG`).
- ✅ **scrypt 해시 로그인**: `users` 테이블(고정 위치 — SQLite는 `data/_auth.db`, Postgres는
  `public` 스키마) + `create_user`/`verify_password`/`user_exists`. 앱 게이트가 로그인/회원가입
  탭으로 분리, 오답 비밀번호는 명확히 거부(예전엔 오타로도 새 빈 계정이 조용히 생성됐음).
- ✅ **기존 사용자 데이터 100% 보존**: 계정별 데이터 격리 키(`user_db_path`) 공식을 **그대로
  유지**(`hash(이름.lower()+"::"+비밀번호)`) — 새 `users` 인증 테이블은 이 키와 별개로 추가된
  것뿐이라, 기존 사용자가 예전과 같은 이름+비번으로 "회원가입"만 다시 하면 기존 파일에 자동
  재연결됨(별도 마이그레이션 스크립트 불필요, 자연스러운 흐름으로 해결).
- ✅ **git 저장소 초기화 + 첫 커밋** 완료(`ad74df0`). `.gitignore`가 `*.db`/`data/`/
  `.streamlit/secrets.toml` 제외 확인함(민감정보 미포함). git identity는 사용자가 직접
  `git config --global`로 설정(Gwangsu / an78964@naver.com) — 이 저장소 안내에 따라 진행.
- ⚠️ **알려진 한계**: `export_db_bytes`/`import_db_bytes`(사이드바 백업 버튼)는 SQLite 파일
  직접 백업 방식이라 Postgres 모드에선 비활성(둘 다 조용히 no-op) — Postgres는 이미 영구
  저장이라 필요성 낮음. 정말 필요해지면 pg_dump 또는 JSON 전체 내보내기로 별도 구현 필요.
- ⚠️ **실제 Neon 연결은 미검증**: 실 Neon 계정 발급이 필요해 로컬에서 라이브 접속 테스트는
  못 함. `_PGCursorWrapper`(플레이스홀더 치환)·`_schema_name`(스키마 격리) 로직은 mock으로
  단위검증 완료(test_pg_wrapper_placeholder_translation, test_pg_schema_name_stable). 사용자가
  Neon URL 발급 후 실배포 시 최초 1회 실제 연결 확인 필요.

## 6. 알려진 한계 / 기술부채
- `use_container_width` deprecation 경고(콘솔만, 동작 정상). streamlit 1.58에선 작동.
  추후 `width='stretch'`로 교체 가능.
- **SQLite 모드**(Neon 미설정 시)는 여전히 Streamlit Cloud 재시작 시 휘발 가능 →
  Neon 설정을 강력 권장(README 배포 가이드 2번). 설정하면 이 문제 해결됨.
- **Narrative 버즈/톤은 현재 시점 스냅샷**(위 4-4 참조) — 무료 API 한계로 과거 시계열 대비
  z-score는 계산 불가. 방향성 참고용으로만 사용.
- 과거 임의시점 뉴스는 무료 API로 못 받음 → "추세전환+내러티브"는 **사용자가 기록한 촉매**를
  전환점에 연결하는 방식(의도된 설계). 이제 뉴스 카드에서 원클릭으로 촉매 기록 가능(4-4).
- 사용자 게이트는 **scrypt 해시로 실제 검증**하지만 이메일 찾기·2FA 등 완전한 보안 인증은
  아님. 민감정보 입력 금지 원칙은 유지.
- 루트의 `portfolio.db`는 초기 테스트 잔재(gitignore됨). 신규 구조는 `data/` 사용.
- Postgres 백업/복원 미지원(위 4-3 참조).

## 7-1. 데이터 모델 메모(신규)
- `scenarios` 테이블: symbol PK, JSON `{horizon_years, cases:[{name,color,prob,comment,
  drivers:[{key,kind:eps|revenue|custom,base,cagr,unit,exit_multiple,_id}], _id}]}`.
  `_id`(uuid)는 Streamlit 위젯 키 안정화용 — 저장돼도 무해, 재계산 파생.
- `peers` 테이블: symbol PK, JSON 티커 배열.
- `users` 테이블(고정 위치 — 사용자 스키마/파일과 별개): name PK, pw_hash, pw_salt, created_at.
- 스코어카드의 `metric`(Live 실측)은 **저장 안 함** — autofill_metrics로 매 로드 재계산.
- narrative.py는 **DB 테이블 없음**(전부 순수함수, 매 요청 시 실시간 계산 + st.cache_data 캐싱).
